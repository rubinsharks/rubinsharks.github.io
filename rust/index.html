Rust 소개

Rust를 사용해야 하는 이유

  1. Rust는 빠릅니다.

    1. 현대적 구문을 채용하지만 C와 거의 동일하게 동작하게 설계되었습니다.

    2. 가비지 콜렉터가 없습니다. 대신 소유권 이동을 통해 메모리 안정성을 보장합니다.

    3. C 라이브러리를 바로 사용가능합니다.

  2. 강력한 크로스 컴파일

    1. 거의 모든 플랫폼에서 동작 가능합니다. 심지어 모바일에서도.

    2. 의외로 웹에서 무섭게 연구되는 중입니다.

    3. 툴체인 자체를 티어별로 제공합니다.

  3. 강력한 언어 기능

    1. 최근 언어에서나 나올법한 고차함수, 확장, 축약까지 거의 모든게 있습니다.

    2. 모듈 분리가 매우 쉽고 명확합니다.

    3. 테스트, 벤치마크가 어노테이션만으로 가능하도록 설계되어 있습니다.

  4. 생각보다 괜찮은 고성능 라이브러리

    1. 러스트를 사용하는 그룹이 백엔드 연산을 주로 하기에 관련 라이브러리가 많습니다.

    2. 일단 만들면 성능상 우위이기 때문에 활발히 라이브러리가 제작되고 있습니다.

  5. Cargo라는 좋은 패키지매니저

    1. 딱히 단점을 찾기 힘들정도로 편합니다.

단점

  1. 난이도가 높습니다. 초보자는 그냥 안하는게 나을정도로.

    1. 포인터 구조를 이해해야 프로그래밍이 가능합니다.

    2. 힙이나 스택의 라이프타임을 이해해야 합니다.

    3. 어디서 어떻게 비용이 지불되는지 이해해야 합니다.

    4. 가비지 콜렌터 없이 메모리 안정성을 높이기위한 소유권 이전에 대해 확실히 이해해야 합니다.

  2. 자기참조/순환참조 구조에 취약합니다.

  3. 고성능 대결을 할게 아니라면 똑같은 고세대 언어와 생산성 차이가 납니다.
